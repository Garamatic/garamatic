STEP 7 4 min

Odoo Bridge — The Complex Integration

Odoo Bridge WOW
1. DEDUP LiteDB — "Has this ticket been processed?"
2. AUTH JSON-RPC authenticate() → UID
3. CUSTOMER res.partner search — create if missing
4. INVOICE account.move — [0, 0, {...}] nested create
5. POST action_post — draft → posted
6. PUBLISH invoice.created event via outbox

Why this matters: Odoo CE has no REST API. Every call is JSON-RPC with session auth. The [0, 0, {...}] syntax is real — nested array means "create new relation record."

  • Circuit breaker: Odoo down → bridge backs off, no cascade
  • State machine: LiteDB tracks every processed ticket
  • Poller: every 5 min, detects payment status changes
