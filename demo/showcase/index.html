Demo Flow (~9 min)

0
Intro — Architecture Overview 0:00 — 0:30
Show the Architecture Dashboard with all services running. Introduce the team (Juan Benjumea, Wito De Schrijver, Maarten Görtz, Charlotte Schröer), the project, and the multi-tenant containerized microservices ecosystem.
Open Dashboard
1
Multi-tenant Portal — Burger Melding 0:30 — 2:00
Citizen submits a ticket via the Desgoffe portal. The form adapts to the tenant's branding and language. Behind the scenes, a ticket.created event is published via the Outbox Pattern.
Open Portal Open Ticket Masala RabbitMQ
2
Ticket Lifecycle + GERDA AI 2:00 — 4:00
Login as manager. GERDA AI sidebar auto-analyses each ticket: complexity, priority, tags, and recommended agent. Assignment triggers a ticket.assigned event.
Ticket Masala
3
MCP AI Chat — Tool-Calling Agent 4:00 — 6:00
Open the AI chat inside any ticket. The agent already knows the ticket context. Ask it to create a €200 invoice and it will call your business APIs directly through MCP tools — no wrapper, no abstraction layer.
Open Ticket Masala Agentic Service
4
Invoicing + Odoo ERP 6:00 — 8:00
Resolve a ticket with a billable amount. The Odoo Bridge receives ticket.resolved, creates a customer and invoice via JSON-RPC, and publishes invoice.created. Zero manual work. Tip: Use the pre-seeded ticket Demande de permis — Extension terrasse (Marie Curie), set status to Completed, add resolution notes and billable amount €150. You must be logged in to resolve a ticket.
Odoo ERP Odoo Bridge
5
Event Bus & Email — Closing the Loop 8:00 — 9:00
One event fans out to three queues. RabbitMQ shows the bindings. Mailhog captures every email: ticket created, assigned, resolved, invoice created. No service calls another directly.
RabbitMQ Mailhog
6
Outro — Summary & Links 9:00 — 9:30
Return to the dashboard. All services running. One binary serves four tenants. Services communicate through events, not direct calls. The AI agent has real access to your APIs. Everything ships in a container. The code is open source.

Event Flow Architecture

Citizen Portal Flow
Portal → Gatekeeper → RabbitMQ → Ticket Masala → Mailing → Mailhog
Portal submits ticket → Gatekeeper validates & publishes ticket.created → RabbitMQ fan-out → Ticket Masala stores + Outbox → Mailing sends confirmation → Mailhog captures.
AI + ERP Flow
MCP Chat → Agentic → Ticket Masala → Odoo Bridge → Odoo ERP
User asks AI to create invoice → Agentic calls MCP tools → Ticket Masala updates → Odoo Bridge syncs via JSON-RPC → Invoice posted in Odoo.
RabbitMQ Exchange
event_exchange → agentic_consumer → mailing.queue → odoo-bridge
One message, three consumers, zero coupling. Outbox Pattern guarantees atomic delivery.
