## Anti-Patterns Verdict

**Would this look AI-generated? Yes, in its current state.** Three clear tells:

**Gradient stat cards.** DESIGN.md bans decorative gradients ($6 Don't #1 — "The only allowed gradient is the GERDA AI badge"). Yet `theme.css:197-219` and `site.css:376-402` define 7 gradient stat card variants. These are the #1 visual signature of template dashboards.

**Side-tab border stripes.** The detector found **12 instances** of `border-left: 3-4px` as colored accent stripes across the default theme and all 4 tenants. DESIGN.md §6 Don't #4 bans this explicitly.

**Legacy green color leaks.** Focus rings (`design-system.css:668`, `site.css:795`), sidebar active backgrounds (`site.css:84`), and badge-primary (`site.css:539`) all use `rgba(76, 175, 80, ...)` — the old green brand color, not Signal Indigo. This is an incomplete migration.

**Deterministic scan (detect.mjs):** 15 findings — 12 side-tab (design-system.css, site.css, whitman, liberty, hennessey, desgoffe tenants), 3 layout-transition (site.css), 1 bounce-easing (design-system.css), 1 overused-font (liberty tenant). Detector caught layout transitions and bounce easing that the design review noted independently.

## Overall Impression

A functionally solid app with a coherent layout system, thorough loading states, and complete form validation — undermined by a visual system that fights its own DESIGN.md. The gradient stat cards and side-stripe borders are the biggest gap between what's documented and what's shipped. The "Masala Kitchen" identity hasn't reached the UI yet; the app still wears its Bootstrap heritage on its sleeve. The single biggest opportunity is replacing the gradient stat cards with tonal layering — that one change would eliminate most of the AI-slop tells.

## What's Working

1. **Loading & empty states are thorough.** Skeleton components (`.skeleton-text`, `.skeleton-title`, `.skeleton-card`, `.skeleton-circle`), HTMX indicators with fade transitions, and empty states with icon/title/CTA are complete and well-structured.

2. **Form validation is complete.** `is-valid`/`is-invalid` styles, feedback messages, required field indicators, and aria-invalid handling in the Desgoffe theme. The app handles errors gracefully.

3. **Tenant theming is ambitious.** Whitman (industrial blue), Liberty (dark/cyberpunk), Hennessey (marine serif), Desgoffe (government confectionery) — each tenant has a real identity with distinct color and typography systems. The CSS variable override pattern is the right approach.

## Priority Issues

### [P0] Gradient Stat Cards Violate DESIGN.md
**Files:** `theme.css:197-219`, `site.css:376-402`
**Why it matters:** 7 gradient stat card variants directly violate The Rarity Rule, The Layering Rule, and the decorative gradient ban. Stat cards with `linear-gradient(135deg)` backgrounds are the #1 visual signature of template dashboards and AI-generated UIs. They also include a decorative `::before` semi-transparent circle that adds visual noise.
**Fix:** Replace all `.stat-card` gradients with `background: var(--bg-primary)` and tonal layering. Reserve Signal Indigo for the `.stat-card-value` text only. Remove `box-shadow: var(--shadow-md)` and the decorative `::before` pseudo-element.
**Suggested command:** `$impeccable polish ticket-masala`

### [P0] Heading Font Is Rajdhani, Not Space Grotesk
**File:** `theme.css:67`
**Why it matters:** `--font-family-heading: 'Rajdhani'` — Rajdhani is a condensed sans-serif commonly used in cheap Bootstrap templates. DESIGN.md specifies Space Grotesk (geometric, wider, tech-forward). The wrong font changes the brand personality from confident/quirky to template-condensed. Neither font is actually loaded in the CSS, so the browser falls through to `system-ui, sans-serif`.
**Fix:** Load Space Grotesk via `@import` or Google Fonts `link`, set `--font-family-heading: 'Space Grotesk', system-ui, sans-serif;`.
**Suggested command:** `$impeccable typeset ticket-masala`

### [P1] Legacy Green Brand Color Leaks in 4+ Locations
**Files:** `design-system.css:668`, `site.css:84`, `site.css:539`, `site.css:795`
**Why it matters:** Four locations use `rgba(76, 175, 80, ...)` — the old brand green (`--color-brand-accent: #8BC34A`). Form control focus rings glow green instead of Signal Indigo. Sidebar active backgrounds tint green. `.badge-primary` uses green tones. This is an incomplete migration that fragments the visual system.
**Fix:** Replace all `rgba(76, 175, 80, *)` with `rgba(99, 102, 241, *)`. Adjust opacity to match intent.
**Suggested command:** `$impeccable polish ticket-masala`

### [P1] Side-Tab Border Stripes (12 Instances Detected)
**Files:** `design-system.css:846`, `site.css:957,964`, plus all 4 tenant CSS files
**Why it matters:** DESIGN.md §6 Don't #4 bans "border-left greater than 1px as a colored stripe accent." The detector found 12 instances across all themes. `.ticket-card` uses 4px indigo left border. Toast notifications use 4px colored left borders. Sidebar nav is exempted (it's the standard active-state affordance).
**Fix:** Replace ticket-card colored left border with a background tint or full top-border approach. Replace toast left borders with full background tints or icon-only indicators.
**Suggested command:** `$impeccable polish ticket-masala`

### [P2] Layout Property Animations (3 Instances Detected)
**Files:** `site.css:35,104,1030`
**Why it matters:** Animating `width`, `margin-left` causes layout thrash and jank. The sidebar collapse animates `width` and `margin-left`. The workload bar animates `width`. Use `transform: translateX()` instead.
**Fix:** Replace `transition: width` with `transition: transform` using `translateX()`. Replace `transition: margin-left` similarly.
**Suggested command:** `$impeccable animate ticket-masala`

### [P2] Missing prefers-reduced-motion
**Files:** All CSS — zero instances found
**Why it matters:** WCAG 2.1 AA requires respecting user motion preferences. The pulse animation on `.badge-dispatching` (site.css:1098), skeleton shimmer (design-system.css:790), and toast slide-in (design-system.css:852) all run unconditionally.
**Fix:** Add one block: `@media (prefers-reduced-motion: reduce) { *, *::before, *::after { animation-duration: 0.01ms !important; animation-iteration-count: 1 !important; transition-duration: 0.01ms !important; } }`
**Suggested command:** `$impeccable harden ticket-masala`

### [P2] Unused Gradient Variables + Dark Mode Fragmentation
**Files:** `theme.css:74-80`, `theme.css:137-158`, `site.css:802-868`
**Why it matters:** 7 gradient variables in theme.css are never used (all gradient usages hardcode their values). Dark mode is defined in two files with overlapping values. `site.css:811-817` unnecessarily hardcodes element-level dark overrides that should cascade from variables.
**Fix:** Remove unused gradient variables. Consolidate all dark mode to `theme.css`. Remove element-level dark overrides. Clean site.css:376 duplicate stat-card.primary definition.
**Suggested command:** `$impeccable distill ticket-masala`

## Persona Red Flags

### Alex (Power User) — Dashboard/Admin Context
- **Gradient stat cards force visual parsing.** Alex scans 5+ stat cards daily. Each has a gradient background + shadow + decorative `::before` circle. The visual noise delays cognitive extraction of numeric values. Flat cards would cut scan time by half.
- **No prefers-reduced-motion.** The pulse animation on `.badge-dispatching` runs unconditionally. For a power user who disables animations, this can cause discomfort.
- **28 `!important` declarations** in site.css mean Alex's custom user stylesheet will fight the cascade. Enterprise deployment risk.

### Sam (Accessibility-Dependent) — Screen Reader User
- **No `prefers-reduced-motion` anywhere.** Every animation runs unconditionally. WCAG 2.2 requires respecting motion preferences.
- **Toast system has no visible `aria-live` or `role="alert"` in CSS.** If the Blazor components don't add these, dynamic updates are silent to screen readers.
- **Gradient stat cards** use white text on colored gradients — contrast varies by gradient segment and is not uniformly guaranteed across the spectrum.
- **`aria-invalid`** is handled in Desgoffe tenant but missing from the base theme. Should be a base pattern.
- **No forced-colors media query** for Windows High Contrast mode.

## Minor Observations

- **`site.css:376-378` and `site.css:392-394`** define `.stat-card.primary` twice with the same gradient — redundant code.
- **GERDA badge gradient** is defined in 3 places: `theme.css:222-227`, `design-system.css:599-608`, and likely in markup. Consolidate.
- **`site.css:74`** sidebar link uses `border-left: 3px solid transparent` as layout placeholder. The 3px visual shift on active/inactive state navigation causes layout jump. Use `outline` or inset `box-shadow` instead.
- **Border-radius drift**: DESIGN.md says 8px for buttons/inputs. `site.css:741` overrides `.btn` to 6px, `site.css:788` overrides `.form-control` to 6px.
- **Toast animation** (`design-system.css:852`) uses `translateX(500px)` with no `will-change: transform` — potential jank on low-end devices.
- **Tenant fonts block rendering**: each tenant loads a separate Google Font via `@import` without `font-display: swap`, causing FOIT/blank text on slow connections.
- **Dispatch header** (`site.css:948`) uses a decorative indigo gradient — banned by DESIGN.md. Use flat indigo background.
- **`.icon-circle`** (`site.css:1192-1200`) is defined but never referenced in any component — dead code or markup-only.
- **JetBrains Mono** is not loaded anywhere — ticket IDs fall back to system monospace.
